Principios Programacion Orientada a Objetos.

La programacion orientada a objetos es uno de los paradigmas de programacion mas populares.

una de las claves del exito del enfoque de la OOP en que nos permite tener un diseño modular y reutilizacion de codigo.

la programacion orientada a objetos se basa en dos requisitos:

* Su capacidad para modelar un problema a través de objetos. Para satisfacer el primer requisito, el lenguaje debe permitir que el desarrollador utilice los objetos de forma real y definiendo las relaciones.

* El apoyo que algunos principios le otorgan a la modularidad y reutilizacion de codigo.

  • Asociación: esta es la capacidad del objeto para referir otro objeto independiente.
  • Agregación: esta es la capacidad del objeto para incrustar uno o más independientes objetos.
  • Composición: esta es la capacidad del objeto para incrustar uno o más dependientes objetos.

    • El segundo requisito se cumple si admite los siguientes principios:

  • Encapsulacion: Esto nos permite concentrarno en una sola entidad de datos con su codigo.
  • Herencia: Esto nos permite que un onjeto adquiere caracteristicas de uno o más objetos.
  • Polimosfismo: Nos permite procesar onjetos de manera diferente.

    Abstraccion y soporte de modelado.

    El primer requisito es el apoyo a modelar un problema a través de objetos. JavaScript admite objetos, pero aquí deberíamos determinar si son compatibles para poder modelar la realidad.

    Necesitamos un modelo porque es una simplificación de realidad, nos permite reducir la complejidad ofreciendo una visión desde una perspectiva particular y nos ayuda a razonar sobre una relación entre entidades.

    Esto es una relacion entre objetos pero cada uno es independiente entre si. Nos da a entender que un objeto puede existir sin otro.

    Encapsulación.

    La encapsulacion es la capacidad de capturar en una entidad ambos datos ocultando la complejidad de la implementacion del mismo. Este es un principio muy poderoso, a menudo encontrado en el mundo real que nos permite usar un objeto sin saber cómo funciona internamente trabajos.

    var empresa = {  nombre: "ACME Inc.",  empleados: [],  sortEmployeesByName: function () {...} };

    Si necesitamos obtener una lista ordenada de empleados de empresa, simplemente necesitamos saber que el método sortEmployeesByName () realiza esta tarea. No necesitamos saber cómo funciona este método, qué algoritmo implementos.
